## 📋 右键菜单功能详解

### 1. 💾 释放工作集

**功能**: 调用 EmptyWorkingSet API  
**效果**: 
- ✅ 释放物理内存（工作集）
- ❌ 不减少已提交内存
- ✅ 进程继续运行
- ✅ 数据不丢失

**适用场景**:
- 物理内存不足
- 系统卡顿换页频繁
- 需要短期缓解内存压力

**操作流程**:
1. 右键点击目标进程
2. 选择"💾 释放工作集"
3. 确认对话框（说明效果）
4. 查看释放结果

**预期效果**:
```
释放前: 已提交 820 MB | 工作集 827 MB
释放后: 已提交 820 MB | 工作集 10-20 MB
```

---

### 2. ⚠️ 终止进程（红色警告）

**功能**: 强制终止进程  
**效果**:
- ✅ 真正释放已提交内存
- ✅ 真正释放物理内存
- ❌ 进程被关闭
- ⚠️ 数据丢失（未保存）

**适用场景**:
- 已提交内存接近上限
- 确认可以关闭程序
- 数据已保存或不重要

**操作流程**:
1. 右键点击目标进程
2. 选择"⚠️ 终止进程"
3. 第一次确认（详细警告）
4. 第二次确认（最终确认）
5. 查看释放结果

**安全机制**:
- ✅ 系统进程完全禁止
- ✅ 双重确认防误操作
- ✅ 详细的风险说明

**预期效果**:
```
释放前: 已提交 820 MB | 工作集 827 MB
释放后: 进程关闭，系统已提交内存 -820 MB
```

---

### 3. 🧪 实验性释放（橙色警告）

**功能**: VirtualFreeEx 尝试释放内存  
**效果**:
- ⚠️ 可能减少已提交内存（10-30%）
- ⚠️ 可能导致进程崩溃（50%+）
- ⚠️ 可能部分数据损坏
- ⚠️ 成功率很低

**适用场景**:
- 仅供研究和学习
- 不建议日常使用
- 如需可靠释放，请用"终止进程"

**操作流程**:
1. 右键点击目标进程
2. 选择"🧪 实验性释放"
3. 第一次警告（说明风险）
4. 查看内存扫描结果
5. 选择释放模式（保守/平衡）
6. 第三次最终确认
7. 查看详细结果

**释放模式**:
- **保守模式**: 风险 ≤ 20（最安全，但可能没有可释放区域）
- **平衡模式**: 风险 ≤ 40（可能释放更多，但崩溃风险高）

**预期效果**:
```
扫描结果:
- 低风险: 0-100 MB（可以安全释放）
- 中风险: 100-800 MB（可能导致崩溃）
- 高风险: 不会尝试释放

实际效果:
- 最佳情况: 释放部分内存，进程继续运行
- 常见情况: 进程崩溃
- 最差情况: 释放 0，或崩溃
```

## 🎯 使用场景推荐

### 场景 1：物理内存不足，但已提交内存充足

**症状**: 系统卡顿，换页频繁  
**解决方案**: 右键 → 💾 释放工作集  
**效果**: 释放物理内存，改善性能

---

### 场景 2：已提交内存接近上限

**症状**: 新程序无法启动，提示内存不足  
**解决方案**: 
1. 右键高内存进程 → 📋 查看详情
2. 确认建议关闭的进程
3. 右键 → ⚠️ 终止进程
4. 查看系统已提交内存是否下降

**效果**: 真正释放已提交内存

---

### 场景 3：研究和学习（实验性）

**症状**: 想了解 Windows 内存管理  
**解决方案**:
1. 右键任意进程 → 📋 查看详情
2. 分析内存布局和风险评估
3. （可选）尝试 🧪 实验性释放
4. 观察效果和学习

**效果**: 深入理解内存管理机制

## 💡 快捷操作技巧

### 技巧 1：快速查看建议关闭的进程
- 查看"状态"列
- 找到标记"💡 建议关闭"的进程
- 这些通常是内存占用 > 500 MB 的非系统进程

### 技巧 2：按类型筛选
- 注意"类型"列
- 浏览器：通常占用多，可以关闭部分标签页
- 开发工具：可能是 IDE，确认后可关闭
- 系统进程：不要动

### 技巧 3：先查看详情再操作
- 不确定时，先右键 → 📋 查看详情
- 根据操作建议决定使用哪个功能

---

## ⚠️ 注意事项

1. **系统进程**：标记为"🔒 受保护"的进程不能操作
2. **数据保存**：终止进程前先保存数据
3. **实验性功能**：仅供研究，不推荐日常使用
4. **多重确认**：危险操作都有多重确认，请仔细阅读
